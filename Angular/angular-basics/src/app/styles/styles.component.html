Style Component

If you want to style the host element the only way you can do it is using the ":host" pseudo selector

You can also use the syntax ":host(.selected)" which will apply the styles on host only if the host element has the class selected

The following syntax:

      :host-context(.theme-light) h2 {{ '{' }}
        background-color: #eef;
        {{ '}' }}

applies a background-color style to all h2 elements inside the component, only if some ancestor element has the CSS class theme-light.

We also control how the style rules will render in our html using the encapsulation property in our component decorator. There are 3 possible values:

  • ViewEncapsulation.Native that uses the native browser shadow DOM implementation

  • ViewEncapsulation.Emulated which is the default implementation in Angular and emulates the behavior of shadow DOM by preprocessing (and renaming) the CSS code to effectively scope the CSS to the component's view.

  • ViewEncapsulation.None Angular does no view encapsulation. Angular adds the CSS to the global styles. The scoping rules, isolations, and protections discussed earlier don't apply. This is essentially the same as pasting the component's styles into the HTML.
